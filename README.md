# Compiler-Design

This repository contains my practice programs for understanding Compiler Design concepts as part of my 6th Semester Compiler Design Lab.
The goal of this repository is to learn and implement the core phases of a compiler using LEX, with clear, experiment-wise programs.
## Compiler Design Lab Programs

- Experiment 1

Simulation of a Finite State Automata (FSA) to recognize control statements using LEX.
This program uses LEX rules to identify control structures such as if, else, while, for, etc.

- Experiment 2

Simulation of a Finite State Machine to distinguish Integers, Real Numbers, and Exponent Numbers.
This experiment classifies numeric inputs based on their format using FSM logic implemented in LEX.

- Experiment 3

LEX program to recognize and return tokens of a C-like language.
This program identifies tokens such as keywords, identifiers, operators, constants, and special symbols.

- Experiment 4

LEX program to eliminate Left Recursion and perform Left Factoring on a given grammar.
This experiment focuses on grammar transformation techniques used in compiler design to make grammars suitable for top-down parsing:

- Experiment 5

Program to find First and Follow sets
This experiment finds out the first and follow sets of any given production rules.

- Experiment 6

Program to design LL parser.
This program scans the statement and tells whether the given statement is syntactically correct or not.
